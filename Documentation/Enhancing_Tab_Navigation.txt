Enhancing Your App’s Content with Tab Navigation
================================================

Keep your app content front and center while providing quick access to navigation using the tab bar.

Overview
--------
Starting in iPadOS 18, the tab bar appears at the top of the screen, floating over your content instead of at the bottom. This design creates an immersive, full-screen browsing experience.

Tab bars provide access to top-level navigation in your app. However, too many tabs can make it hard for people to locate content. Implementing a sidebar makes it easier to navigate a detailed information hierarchy.

Creating a TabView
------------------
You can create a TabView with an explicit selection binding using the following initializer:

    TabView(selection:content:)

To add a tab, initialize a Tab using:

    Tab(_:systemImage:value:content:)

The selection type of the TabView matches the value type of its contained tabs.

Example:

    @State private var selectedTab: Tabs = .watchNow

    var body: some View {
        TabView(selection: $selectedTab) {
            Tab("Watch Now", systemImage: "play", value: .watchNow) {
                WatchNowView()
            }
            // More tabs...
        }
    }

    enum Tabs: Equatable, Hashable, Identifiable {
        case watchNow
        case library
        case new
        case favorites
        case search
    }

Search Tabs
-----------
You can create a search tab using the role parameter:

    Tab(value: .search, role: .search) {
        // View for search
    }

When using symbol images for tabs, use the outline variant. The system automatically selects the filled version when appropriate.

A tab with the `.search` role gets:
- Default title “Search”
- Default system symbol (magnifying glass)
- Automatically pinned behavior

Pinned tabs appear at the trailing edge of the tab bar (on the right in left-to-right languages, left in right-to-left).

Building a Hierarchy in a TabView
---------------------------------
Use `TabSection` to declare a secondary tab hierarchy within a TabView.

Example:

    TabView(selection: $selectedTab) {
        Tab("Watch Now", systemImage: "play", value: .watchNow) {
            WatchNowView()
        }

        TabSection {
            Tab("Cinematic Shots", systemImage: "list.and.film", value: .collections(.cinematic)) {
                // ...
            }
        } header: {
            Label("Collections", systemImage: "folder")
        }
    }

You can define enums to organize nested content:

    enum Tabs: Equatable, Hashable, Identifiable {
        case watchNow
        case search
        case collections(Category)
        case animations(Category)
    }

    enum Category: Equatable, Hashable, Identifiable, CaseIterable {
        case cinematic
        case forest
        case sea
        // ...
    }

ForEach Example:

    TabSection {
        ForEach(Category.collectionsList) { collection in
            Tab(collection.name, systemImage: collection.icon, value: Tabs.collections(collection)) {
                // ...
            }
        }
    } header: {
        Label("Collections", systemImage: "folder")
    }

Making the Tab Bar Adaptable
----------------------------
To let people toggle between the sidebar and tab bar, use the sidebarAdaptable tab view style.

Example:

    TabView(selection: $selectedTab) {
        // Tabs
    }
    .tabViewStyle(.sidebarAdaptable)

Behavior by platform:
- In iPadOS, appears as a top tab bar that can become a sidebar.
- In compact size, appears as a bottom tab bar.

By default, horizontal ScrollView content scrolls under the sidebar. Prevent this with:
    
    .clipped(antialiased: true)

or

    .clipShape(_:style:)

Enabling Customization
----------------------
You can let users customize their tab bar and sidebar order.

Example:

    @AppStorage("sidebarCustomizations") var tabViewCustomization: TabViewCustomization
    @State private var selectedTab: Tabs = .watchNow

    var body: some View {
        TabView(selection: $selectedTab) {
            Tab("Watch Now", systemImage: "play", value: .watchNow) {
                WatchNowView()
            }
            .customizationID(Tabs.watchNow.customizationID)

            // More tabs...
        }
        .tabViewCustomization($tabViewCustomization)
    }

You can disable customization for critical tabs:

    Tab("Watch Now", systemImage: "play", value: .watchNow) {
        WatchNowView()
    }
    .customizationBehavior(.disabled, for: .sidebar, .tabBar)

Customizations allow people to:
- Drag and drop tabs to add/remove them
- Hide non-essential tabs
- Reorder tabs in both the sidebar and tab bar

Persisting Customizations
-------------------------
Persist using `AppStorage` with an identifier for a `TabViewCustomization` variable.  
Attach `.tabViewCustomization(_:)` to your TabView.  
Add `.customizationID(_:)` to each tab.

Limiting Tab Count
------------------
If there are too many tabs, the system collapses overflow items and enables scrolling.

It’s recommended to limit tabs so they all fit on-screen.  
Use `.defaultVisibility(_:, for:)` to manage tab visibility:

    .defaultVisibility(.hidden, for: .tabBar)

For tabs that should appear only in the sidebar:

    .sidebarOnly

Summary
-------
Tab bars provide quick navigation to key sections while keeping content central.  
The sidebarAdaptable style combines both tab and sidebar experiences in one navigation model.  
Customization lets people personalize navigation while maintaining clarity and focus in iPad and macOS apps.
